# FEATURES — vtmdev/dotenv-editor (fork)
Source: github.com/vtmdev/dotenv-editor · no version tag (branch-alias 3.x-dev) · MIT

Fork of jackiedo/laravel-dotenv-editor. Diffs from jackiedo: almost entirely cosmetic — docblock `@return`/tag reordering, indentation/whitespace changes, `\Exception` → imported `Exception` in the seven Exception classes, `'/.env'` → `'/' . '.env'`. The one real behavior change: `stringToType()` is removed from the shared `CreateCommandInstanceTrait` and copy-pasted into each console command, and `DotenvSetKeyCommand` now passes the `value` argument through `stringToType()`, so `dotenv:set-key KEY true|false|null` writes a typed value instead of the literal string. Still ships ParserV1/V2/V3 and the full jackiedo parser-selection logic; Laravel capped at `^10` (with a malformed `7.0` constraint missing its caret). Net-new public API: none. Tests: N (no tests/, no require-dev). Abandoned/stale: stale/abandoned — looks like an old IDE/CS-fixer reformat of jackiedo with one CLI tweak; no Laravel 11+ support, no tests, no version tag, third-party authorship (Vantom Developer). Least useful of the four.
